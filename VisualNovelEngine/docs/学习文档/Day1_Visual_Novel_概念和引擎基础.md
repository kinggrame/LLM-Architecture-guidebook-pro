# 第1天：什么是 Visual Novel（视觉小说）？

## 1.1 什么是 Visual Novel？

Visual Novel（视觉小说），简称 VN，也被称为 Galgame（恋爱游戏），是一种以叙事为主的电子游戏类型。

### 视觉小说的特点：
- **以文字叙述为主**：玩家主要通过阅读故事文本来了解剧情
- **图片和音效**：配合剧情显示背景图片、角色立绘、播放音乐
- **分支选择**：玩家需要做出选择，不同选择会导致不同剧情发展
- **互动性**：玩家不是被动观看，而是参与剧情推进

### 经典视觉小说示例：
- 《CLANNAD》
- 《命运石之门》
- 《Fate/stay night》
- 《Summer Pockets》

---

## 1.2 什么是游戏引擎？

**游戏引擎**是一套用于开发游戏的软件框架，它提供了：
- 图形渲染（显示图片、动画）
- 音频处理（播放音乐、音效）
- 资源管理（加载和存储游戏素材）
- 场景管理（切换游戏画面）
- 用户输入（键盘、鼠标操作）
- 脚本系统（编写游戏剧情）

**常见的游戏引擎：**
- Unity（最流行）
- Unreal Engine（虚幻引擎）
- Godot（开源免费）
- RPG Maker（专门做RPG）

---

## 1.3 我们的项目：Visual Novel Engine

这个项目是一个**专门为视觉小说设计的轻量级游戏引擎**。

### 核心功能：
1. **脚本系统**：用简单的文本格式编写剧情
2. **角色立绘**：显示角色图片
3. **背景切换**：切换场景背景
4. **音频播放**：背景音乐、音效
5. **分支选择**：玩家选择剧情走向
6. **场景管理**：管理不同的游戏画面

## 1.4 引擎的整体架构

```
┌─────────────────────────────────────────────────┐
│                  主引擎 (Engine)                 │
│  协调所有子系统，确保各部分正常工作              │
└─────────────────────────────────────────────────┘
          │        │        │        │
    ┌─────┴────┐┌─┴──────┐┌──┴────┐┌──┴─────┐
    │ 渲染引擎  ││脚本引擎 ││场景管理││资源管理│
    │          ││         ││       ││        │
    │ 显示图片  ││解析脚本 ││切换场景││加载素材│
    │ 绘制UI   ││执行命令 ││管理状态││缓存资源│
    └──────────┘└─────────┘└───────┘└────────┘
          │
    ┌─────┴─────┐
    │ 事件管理器 │
    │ 处理输入   │
    └──────────┘
```

### 各组件职责：

| 组件 | 职责 |
|------|------|
| **Engine（主引擎）** | 统筹协调，启动和关闭所有子系统 |
| **RenderEngine（渲染引擎）** | 在屏幕上绘制图片、文字、UI |
| **ScriptEngine（脚本引擎）** | 读取和执行剧情脚本 |
| **SceneManager（场景管理器）** | 管理游戏的不同场景/画面 |
| **ResourceManager（资源管理器）** | 加载和缓存图片、音频等素材 |
| **EventManager（事件管理器）** | 处理玩家的键盘、鼠标输入 |
