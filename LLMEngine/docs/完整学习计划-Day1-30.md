# 第一阶段：推理引擎基础 (Day 1-7)

## Day 1: LLM 推理概述

### 1.1 推理 vs 训练

| 阶段 | 特点 |
|------|------|
| 训练 | 一次 forward + backward，大量数据 |
| 推理 | 仅 forward，延迟敏感，批量处理 |

### 1.2 推理挑战

1. **延迟**: 用户等待时间，影响体验
2. **吞吐量**: 每秒处理请求数
3. **内存**: 模型权重+KV Cache 占显存
4. **成本**: GPU/CPU 资源消耗

### 1.3 推理流程

```
用户请求 → 分词 → 前向传播 → 采样 → 解码 → 后处理 → 返回
     ↓
   Token by Token (自回归)
```

### 1.4 推理引擎架构

```
┌─────────────────────────────────────────────────────┐
│                  LLM Engine                         │
├─────────────────────────────────────────────────────┤
│  ┌─────────┐  ┌─────────┐  ┌─────────┐             │
│  │Tokenizer│  │ Model   │  │ Sampler │             │
```

```rust
pub struct ModelLoader {
    path: String,
    model_type: ModelType,
    quantization: QuantizationType,
}

impl ModelLoader {
    /// 加载模型
    pub fn load(&self) -> Result<LoadResult> {
        let path = Path::new(&self.path);
        
        // 读取文件头检测格式
        let mut file = File::open(path)?;
        let mut header = [0u8; 12];
        file.read(&mut header)?;
        
        // 解析 GGUF
        if &header[0..4] == b"gguf" {
            return self.load_gguf_format();
        }
        
        // 其他格式...
        self.load_safetensors()
    }
    
    /// 加载 GGUF 格式
    fn load_gguf_format(&self) -> Result<LoadResult> {
        // 1. 读取魔数
        let mut reader = BufReader::new(File::open(&self.path)?);
        
        // 2. 读取模型类型
        let model_type = self.read_string(&mut reader)?;
        
        // 3. 读取配置
        let config = self.read_config(&mut reader)?;
        
        // 4. 读取权重
        let weights = self.load_weights(&mut reader, &config)?;
        
        Ok(LoadResult { config, weights, ... })
    }
    
    /// 加载权重
    fn load_weights(&self, reader: &mut BufReader<File>, config: &ModelConfig) 
        -> Result<ModelWeights> {
        let mut weights = ModelWeights {
            token_embedding: self.read_tensor(reader, config.vocab_size, config.embedding_dim)?,
            layers: Vec::new(),
            final_layernorm: self.read_tensor(reader, 1, config.embedding_dim)?,
            lm_head: self.read_tensor(reader, config.vocab_size, config.embedding_dim)?,
        };
        
        // 加载每一层
        for _ in 0..config.num_layers {
            weights.layers.push(self.load_layer(reader, config)?);
        }
        
        Ok(weights)
    }
}
```

### 3.3 代码实现

```rust
// src/tokenizer/mod.rs

pub struct BPETokenizer {
    vocab: HashMap<String, usize>,
    inverse_vocab: Vec<String>,
    merges: Vec<(String, String)>,
    config: TokenizerConfig,
}

impl Tokenizer for BPETokenizer {
    fn encode(&self, text: &str) -> Result<Vec<usize>> {
        let mut tokens = Vec::new();
        
        // 添加 BOS
        if let Some(bos_id) = self.config.bos_token_id {
            tokens.push(bos_id);
        }
        
        // BPE 编码
        for word in text.split_whitespace() {
            let word_tokens = self.bpe_encode(word);
            for token in word_tokens {
                if let Some(&id) = self.vocab.get(&token) {
                    tokens.push(id);
                } else {
                    tokens.push(self.config.unk_token_id.unwrap_or(0));
                }
            }
        }
        
        // 添加 EOS
        if let Some(eos_id) = self.config.eos_token_id {
            tokens.push(eos_id);
        }
        
        Ok(tokens)
    }
    
    fn decode(&self, ids: &[usize]) -> Result<String> {
        let mut text = String::new();
        
        for id in ids {
            if let Some(token) = self.inverse_vocab.get(*id) {
                text.push_str(token);
            }
        }
        
        Ok(text)
    }
}
```

### 4.1 前向传播

```rust
// src/inference/mod.rs

/// 推理引擎
pub struct InferenceEngine<M: Model> {
    model: Arc<M>,
    config: InferenceConfig,
    kv_cache: Option<KVCache>,
}

impl<M: Model> InferenceEngine<M> {
    /// 生成文本
    pub fn generate(&mut self, prompt: &[usize], max_new_tokens: usize) 
        -> Result<Vec<usize>> {
        let mut all_tokens = prompt.to_vec();
        
        for _ in 0..max_new_tokens {
            // 前向传播
            let logits = self.forward(&all_tokens)?;
            
            // 采样下一个 token
            let next_token = self.sample(&logits[all_tokens.len() - 1]);
            
            // 检查结束
            if next_token == self.model.config().eos_token_id {
                break;
            }
            
            all_tokens.push(next_token);
        }
        
        Ok(all_tokens[prompt.len()..].to_vec())
    }
    
    fn forward(&mut self, input_ids: &[usize]) -> Result<Array2<f32>> {
        let position_ids: Vec<usize> = (0..input_ids.len()).collect();
        self.model.forward(input_ids, Some(&position_ids), self.kv_cache.as_ref())
    }
}
```

## Day 5: 采样策略

### 5.1 Greedy Sampling

始终选择概率最高的 token。

```rust
fn greedy_sample(logits: &[f32]) -> usize {
    logits.iter()
        .enumerate()
        .max_by(|(_, a), (_, b)| a.partial_cmp(b).unwrap())
        .map(|(i, _)| i)
        .unwrap()
}
```

### 5.2 Top-k Sampling

只从概率最高的 k 个 token 中选择。

```rust
fn top_k_sample(logits: &[f32], k: usize) -> usize {
    // 排序
    let mut indices: Vec<usize> = (0..logits.len()).collect();
    indices.sort_by(|&i, &j| logits[j].partial_cmp(&logits[i]).unwrap());
    
    // 取前 k 个
    let top_k_logits: Vec<f32> = indices.iter()
        .take(k)
        .map(|&i| logits[i])
        .collect();
    
    // Softmax
    let sum: f32 = top_k_logits.iter().map(|x| x.exp()).sum();
    let probs: Vec<f32> = top_k_logits.iter().map(|x| x.exp() / sum).collect();
    
    // 加权随机选择
    let r = rand::random::<f32>();
    let mut cumsum = 0.0;
    for (i, &prob) in probs.iter().enumerate() {
        cumsum += prob;
        if r <= cumsum {
            return indices[i];
        }
    }
    
    indices[k-1]
}
```

### 5.3 Top-p (Nucleus) Sampling

从累积概率达到 p 的最小集合中选择。

```rust
fn top_p_sample(logits: &[f32], p: f32) -> usize {
    // Softmax
    let max_logit = logits.iter().cloned().fold(f32::NEG_INFINITY, f32::max);
    let exp_logits: Vec<f32> = logits.iter()
        .map(|l| (l - max_logit).exp())
        .collect();
    let sum: f32 = exp_logits.iter().sum();
    let probs: Vec<f32> = exp_logits.iter().map(|l| l / sum).collect();
    
    // 排序
    let mut indexed: Vec<(usize, f32)> = probs.iter()
        .enumerate()
        .map(|(i, &p)| (i, p))
        .collect();
    indexed.sort_by(|a, b| b.1.partial_cmp(&a.1).unwrap());
    
    // 累积概率
    let mut cumsum = 0.0;
    let cutoff = indexed.iter()
        .take_while(|(_, prob)| {
            cumsum += prob;
            cumsum < p
        })
        .count()
        .max(1);
    
    // 从 cutoff 中采样
    let selected: Vec<(usize, f32)> = indexed[..cutoff].to_vec();
    let selected_sum: f32 = selected.iter().map(|(_, p)| p).sum();
    let r = rand::random::<f32>() * selected_sum;
    
    cumsum = 0.0;
    for (idx, prob) in selected {
        cumsum += prob;
        if r <= cumsum {
            return idx;
        }
    }
    
    selected.last().map(|(i, _)| *i).unwrap()
}
```

```rust
        for logit in logits.iter_mut() {

}
```

### 5.5 重复惩罚

```rust
fn apply_repeat_penalty(logits: &mut [f32], prev_tokens: &[usize], penalty: f32) {
    for &token in prev_tokens {
        if logits[token] > 0.0 {
            logits[token] /= penalty;
        } else {
            logits[token] *= penalty;
        }
    }
}
```

## Day 6-7: KV Cache 原理

### 6.1 为什么需要 KV Cache？

自回归生成时：
- 每步需要计算所有历史位置的 attention
- 之前计算的 K, V 可以复用

### 6.2 KV Cache 结构

```rust
// src/model/mod.rs

pub struct KVCache {
    /// Key 缓存: [layer, batch, head, seq_len, head_dim]
    keys: Vec<Array3<f32>>,
    /// Value 缓存
    values: Vec<Array3<f32>>,
    seq_len: usize,
    max_seq_len: usize,
}

impl KVCache {
    /// 更新缓存
    pub fn update(&mut self, layer_idx: usize, pos: usize, key: &[f32], value: &[f32]) {
        // 缓存新的 K, V
        for head in 0..self.num_heads {
            for d in 0..self.head_dim {
                self.keys[layer_idx][[head, pos, d]] = key[head * self.head_dim + d];
                self.values[layer_idx][[head, pos, d]] = value[head * self.head_dim + d];
            }
        }
        self.seq_len = pos + 1;
    }
}
```

### 6.3 使用 KV Cache 的注意力

```rust
fn attention_with_cache(
    q: &[f32],
    k_cache: &KVCache,
    v_cache: &KVCache,
    current_pos: usize,
) -> Vec<f32> {
    let mut scores = Vec::new();
    
    // 使用缓存的 K, V
    for pos in 0..current_pos {
        let mut score = 0.0;
        for d in 0..head_dim {
            score += q[d] * k_cache.get_key(layer, head, pos)[d];
        }
        scores.push(score / (head_dim as f32).sqrt());
    }
    
    // Softmax
    softmax(&mut scores);
    
    // 加权求和
    let mut output = vec![0.0; head_dim];
    for (pos, &weight) in scores.iter().enumerate() {
        for d in 0..head_dim {
            output[d] += weight * v_cache.get_value(layer, head, pos)[d];
        }
    }
    
    output
}
```

## Day 8-9: 量化技术

### 8.1 量化概述

| 类型 | 位宽 | 压缩 | 精度损失 |
|------|------|------|----------|
| FP32 | 32 | 1× | - |
| FP16 | 16 | 2× | 小 |
| BF16 | 16 | 2× | 更小 |
| INT8 | 8 | 4× | 中 |
| INT4 | 4 | 8× | 大 |
| NF4 | 4 | 8× | 更小 |

### 8.2 量化原理

```
量化: x_q = round(x / scale)
反量化: x = x_q × scale
```

### 8.3 对称量化

```
scale = max(|x|) / 127
x_q = round(x / scale)
x_q ∈ [-127, 127]
```

### 8.4 代码实现

```rust
// src/quantization/mod.rs

/// INT8 对称量化
pub fn quantize_int8(weights: &Array2<f32>) -> (Vec<i8>, f32) {
    let max_abs = weights.iter()
        .map(|x| x.abs())
        .fold(0.0f32, f32::max);
    
    let scale = max_abs / 127.0;
    let quantized: Vec<i8> = weights.iter()
        .map(|&x| (x / scale).round() as i8)
        .collect();
    
    (quantized, scale)
}

/// INT8 反量化
pub fn dequantize_int8(quantized: &[i8], scale: f32) -> Vec<f32> {
    quantized.iter()
        .map(|&x| x as f32 * scale)
        .collect()
}
```

### 8.5 GPTQ 量化

按组量化，减少精度损失。

```rust
/// GPTQ 量化
fn gptq_quantize(weights: &Array2<f32>, group_size: usize) -> QuantizedWeight {
    let (rows, cols) = (weights.shape()[0], weights.shape()[1]);
    let num_groups = cols / group_size;
    
    let mut quantized_data = Vec::with_capacity(rows * cols);
    let mut scales = Vec::with_capacity(num_groups * rows);
    
    for g in 0..num_groups {
        // 获取当前组
        let group_weights = weights.slice(s![.., g*group_size..(g+1)*group_size]);
        
        // 计算组 scale
        let max_abs = group_weights.iter()
            .map(|x| x.abs())
            .fold(0.0f32, f32::max);
        let scale = max_abs / 127.0;
        
        // 量化
        for col in 0..group_size {
            for row in 0..rows {
                let val = weights[[row, g*group_size + col]];
                let q = (val / scale).round() as i8;
                quantized_data.push(q);
            }
            scales.push(scale);
        }
    }
    
    QuantizedWeight { data: quantized_data, scales }
}
```

## Day 10-11: 批处理优化

### 10.1 静态批处理

固定批量大小，同时处理多个请求。

```rust
pub fn batch_generate(&mut self, prompts: &[Vec<usize>], max_len: usize) 
    -> Vec<Vec<usize>> {
    // 填充到相同长度
    let padded: Vec<Vec<usize>> = prompts.iter()
        .map(|p| {
            let mut p = p.clone();
            p.resize(max_len, 0);  // pad
            p
        })
        .collect();
    
    // 批量推理
    let batch_size = prompts.len();
    let mut outputs = Vec::new();
    
    // 前向传播一次
    let logits = self.forward_batch(&padded)?;
    
    // 解码
    for i in 0..batch_size {
        outputs.push(self.decode(&logits[i]));
    }
    
    outputs
}
```

### 10.2 连续批处理 (Continuous Batching)

请求完成立即加入新请求，最大化 GPU 利用率。

```
Time →
Request 1: [====]
Request 2:        [======]
Request 3:             [=====]
Request 4:                  [===]
```

### 10.3 代码实现

```rust
struct BatchScheduler {
    max_batch_size: usize,
    max_tokens: usize,
}

impl BatchScheduler {
    fn schedule(&mut self, mut queue: Vec<Request>) -> Vec<Request> {
        let mut batch = Vec::new();
        let mut total_tokens = 0;
        
        while !queue.is_empty() && batch.len() < self.max_batch_size {
            let req = queue.remove(0);
            if total_tokens + req.prompt_len < self.max_tokens {
                batch.push(req);
                total_tokens += req.prompt_len;
            } else {
                queue.push(req);
                break;
            }
        }
        
        batch
    }
}
```

## Day 16-18: Flash Attention

### 16.1 标准注意力复杂度

```
时间: O(N² × d)
空间: O(N²)
N = 序列长度, d = head_dim
```

### 16.2 Flash Attention

利用 GPU 显存层次结构：

```
标准: O(N²) 显存
Flash: O(N) 显存
```

核心思想：分块计算，融合核。

### 16.3 实现

```rust
fn flash_attention(
    q: &[f32], k: &[f32], v: &[f32],
    block_size: usize,
) -> Vec<f32> {
    let n = q.len() / head_dim;
    let mut output = vec![0.0; q.len()];
    
    // 分块计算
    for i in (0..n).step_by(block_size) {
        let q_block = &q[i*head_dim..(i+block_size)*head_dim];
        
        // 计算局部注意力
        let mut block_output = compute_block_attention(q_block, k, v);
        
        // 写回
        output[i*head_dim..(i+block_size)*head_dim].copy_from_slice(&block_output);
    }
    
    output
}
```

## Day 22-25: 高级采样技术

### 22.1 Beam Search

保留 top-k 最优路径。

```rust
fn beam_search(
    &self, 
    prompt: &[usize], 
    beam_width: usize, 
    max_len: usize,
) -> Vec<usize> {
    // 初始化
    let mut beams = vec![(prompt.to_vec(), 0.0f32)];
    
    for _ in 0..max_len {
        let mut candidates = Vec::new();
        
        for (seq, score) in &beams {
            let logits = self.forward(seq)?;
            let top_k = self.get_top_k(&logits, beam_width);
            
            for (token, log_prob) in top_k {
                let new_seq = [seq, &[token]].concat();
                candidates.push((new_seq, score + log_prob));
            }
        }
        
        // 取 top-k
        candidates.sort_by(|a, b| b.1.partial_cmp(&a.1).unwrap());
        beams = candidates.into_iter().take(beam_width).collect();
    }
    
    beams[0].0.clone()
}
```
