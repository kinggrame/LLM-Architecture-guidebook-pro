# 第一阶段：语言模型基础 (Day 1-7)

## Day 1: 什么是语言模型？

### 1.1 概念理解

**语言模型 (Language Model)** 是对文本序列进行概率建模的系统。

```
输入: "今天天气"
输出: "很好" 的概率 vs "很差" 的概率
```

**核心任务：预测下一个词**

```
P(下一个词 | 之前的词)
```

### 1.2 语言模型发展历史

```
1948 - n-gram 模型
    ↓
2013 - Word2Vec (词向量)
    ↓
2017 - Transformer (注意力机制)
    ↓
2018 - GPT (生成式预训练)
    ↓
2020 - GPT-3 (175B 参数)
    ↓
2022 - ChatGPT (RLHF)
```

### 1.3 语言模型类型

| 类型 | 说明 | 代表模型 |
|------|------|----------|
| n-gram | 基于统计的词序列概率 | - |
| RNN/LSTM | 循环神经网络 | - |
| Transformer | 注意力机制 | BERT, GPT |
| 大模型 | 数十亿参数 | GPT-4, LLaMA |

### 1.4 数学表示

给定文本序列 `w1, w2, ..., wn`，语言模型计算：

```
P(w1, w2, ..., wn) = P(w1) × P(w2|w1) × P(w3|w1,w2) × ... × P(wn|w1,...,w(n-1))
```

使用链式法则分解为条件概率的乘积。

## Day 2: 分词器 - Tokenizer

### 2.1 为什么需要分词？

计算机无法直接处理文本，需要将文本转换为数字 ID。

```
文本: "Hello, World!"
↓
Token IDs: [15456, 6, 14957, 999]
```

### 2.2 常见分词方法

**1. 词级分词 (Word-level)**
```
"今天天气很好" → ["今天", "天气", "很好"]
```
问题：词汇表巨大，未登录词(OOV)问题

**2. 字符级分词 (Character-level)**
```
"今天天气很好" → ["今", "天", "天", "气", "很", "好"]
```
问题：序列太长，语义信息丢失

**3. 子词分词 (Subword)**
```
"unhappiness" → ["un", "happiness"]
```
平衡词汇表大小和序列长度

### 2.3 BPE (Byte Pair Encoding)

**核心思想**：频繁出现的字符对合并为新 token

```
Step 1: 初始字符
a b b b a b a

Step 2: 统计字符对频率
"ab" = 3 次, "bb" = 2 次, "ba" = 2 次

Step 3: 合并 "ab"
a (ab) (ab) b a b a

Step 4: 继续合并
a (ab)(ab) (ab) a
```

### 2.4 代码实现

```rust
// src/tokenizer/bpe.rs - BPE 分词器核心

/// BPE 分词器结构
pub struct BPETokenizer {
    vocab: HashMap<String, usize>,      // token -> id
    reverse_vocab: HashMap<usize, String>, // id -> token
    merges: Vec<(String, String)>,     // 合并规则
}

impl BPETokenizer {
    /// 编码：文本 -> token IDs
    pub fn encode(&self, text: &str) -> Vec<usize> {
        // 1. 字符级分词
        let mut tokens: Vec<String> = text.chars()
            .map(|c| c.to_string())
            .collect();
        
        // 2. 应用合并规则
        for (first, second) in &self.merges {
            let mut new_tokens = Vec::new();
            let mut i = 0;
            while i < tokens.len() {
                if i + 1 < tokens.len() 
                    && &tokens[i] == first 
                    && &tokens[i+1] == second {
                    new_tokens.push(format!("{}{}", first, second));
                    i += 2;
                } else {
                    new_tokens.push(tokens[i].clone());
                    i += 1;
                }
            }
            tokens = new_tokens;
        }
        
        // 3. 转换为 IDs
        tokens.iter()
            .map(|t| self.vocab.get(t).unwrap_or(&0))
            .copied()
            .collect()
    }
    
    /// 解码：token IDs -> 文本
    pub fn decode(&self, ids: &[usize]) -> String {
        ids.iter()
            .map(|id| self.reverse_vocab.get(id).unwrap())
            .map(|s| s.as_str())
            .collect()
    }
}
```

### 2.5 特殊 Token

| Token | 含义 | 用途 |
|-------|------|------|
| \<PAD\> | 填充 | 批处理 |
| \<UNK\> | 未知 | OOV |
| \<BOS\> | 句子开始 | 序列开始 |
| \<EOS\> | 句子结束 | 序列结束 |
| \<MASK\> | 掩码 | 掩码语言模型 |

## Day 3: 词向量与Embedding

### 3.1 什么是 Embedding？

**Embedding** 是将离散符号映射到连续向量空间的技术。

```
"cat"     → [0.1, -0.3, 0.5, ...]
"dog"     → [0.2, -0.2, 0.4, ...]
"animal"  → [0.15, -0.25, 0.45, ...]
```

### 3.2 为什么需要 Embedding？

1. **语义相似性**：相似词在向量空间中距离近
2. **可计算**：可以计算向量间的相似度
3. **可学习**：可以通过训练调整向量

### 3.3 Word2Vec 简介

**CBOW (Continuous Bag of Words)**
```
上下文: [The, cat, on, mat]
目标词: sat
```

**Skip-gram**
```
目标词: sat
上下文: [The, cat, on, mat]
```

### 3.4 代码实现

```rust
// src/embedding/token_embedding.rs

/// Token 嵌入层
pub struct TokenEmbedding {
    /// 嵌入矩阵 (vocab_size, embedding_dim)
    embedding: Array2<f32>,
    config: EmbeddingConfig,
}

impl TokenEmbedding {
    /// 前向传播：IDs -> 嵌入向量
    pub fn forward(&self, ids: &[usize]) -> Array2<f32> {
        let seq_len = ids.len();
        let mut output = Array2::zeros((seq_len, self.config.embedding_dim));
        
        for (i, &id) in ids.iter().enumerate() {
            // 从嵌入矩阵中取出对应 ID 的向量
            for j in 0..self.config.embedding_dim {
                output[[i, j]] = self.embedding[[id, j]];
            }
        }
        
        output
    }
}
```

### 3.5 位置编码

Transformer 需要知道 token 的位置信息。

**1. 绝对位置编码**

```
PE(pos, 2i) = sin(pos / 10000^(2i/d))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d))
```

**2. 相对位置编码**

考虑 token 之间的相对距离。

**3. RoPE (旋转位置编码)**

LLaMA 使用的位置编码，更适合长序列。

```rust
// src/embedding/position_embedding.rs

/// 位置编码
pub struct PositionEmbedding {
    encoding_type: PositionEncodingType,
    embeddings: Array2<f32>,
}

impl PositionEmbedding {
    /// 创建正弦位置编码
    pub fn new(encoding_type: PositionEncodingType, dim: usize, max_len: usize) -> Self {
        let mut embeddings = Array2::zeros((max_len, dim));
        
        for pos in 0..max_len {
            for i in (0..dim).step_by(2) {
                let freq = (pos as f32 / 10000.0_f32.powf(2.0 * i as f32 / dim as f32));
                embeddings[[pos, i]] = freq.sin();
                if i + 1 < dim {
                    embeddings[[pos, i + 1]] = freq.cos();
                }
            }
        }
        
        Self { encoding_type, embeddings }
    }
}
```

## Day 4: 神经网络基础

### 4.1 神经网络概述

神经网络是受人脑启发的计算模型：

```
输入层 → 隐藏层 → 输出层
```

### 4.2 核心概念

**1. 线性变换**
```
y = Wx + b
```

**2. 激活函数**
- ReLU: f(x) = max(0, x)
- Sigmoid: f(x) = 1/(1+e^(-x))
- Tanh: f(x) = (e^x - e^(-x))/(e^x + e^(-x))

**3. Softmax**
```
softmax(xi) = exp(xi) / Σ exp(xj)
```

### 4.3 前向传播

```rust
fn forward(x: &[f32], w: &[[f32]], b: &[f32]) -> Vec<f32> {
    // y = xW + b
    w.iter()
        .map(|row| {
            row.iter()
                .zip(x.iter())
                .map(|(wi, xi)| wi * xi)
                .sum::<f32>() + b[row.len()]
        })
        .collect()
}
```

### 4.4 反向传播 (简述)

```
损失 -> 输出层 -> 隐藏层 -> 输入层
```

通过梯度下降更新参数，最小化损失函数。

## Day 5: 注意力机制 - 核心！

### 5.1 什么是注意力？

注意力机制让模型"关注"输入的相关部分。

```
源文本: "The cat sat on the mat"
翻译:   "猫 坐在 垫子 上"
```

翻译 "猫" 时，应该关注源文的 "cat"。

### 5.2 注意力公式

**核心公式：**

```
Attention(Q, K, V) = softmax(QK^T / √dk) V
```

**详细解释：**

```
Q (Query): 我要找什么
K (Key): 我有什么
V (Value): 实际的信息内容
```

### 5.3 注意力计算步骤

```
1. Q × K^T → 相似度矩阵
       [q1]     [k1 k2 k3]
       [q2]  ×  [k1 k2 k3]  =  [score11 score12 score13]
       [q3]               [score21 score22 score23]
                                    ↓
2. 除以 √dk (缩放)
                                    ↓
3. Softmax → 权重
                               [0.7 0.2 0.1]
                                    ↓
4. 权重 × V → 输出
```

### 5.4 代码实现

```rust
// src/attention/multi_head_attention.rs

pub struct MultiHeadAttention {
    w_q: Array2<f32>,  // Query 投影
    w_k: Array2<f32>,  // Key 投影
    w_v: Array2<f32>,  // Value 投影
    w_o: Array2<f32>,  // 输出投影
}

impl MultiHeadAttention {
    pub fn forward(&self, x: &Array2<f32>) -> Array2<f32> {
        let d_k = x.shape()[2] / self.num_heads;
        
        // 1. 线性投影
        let q = x.dot(&self.w_q.t());
        let k = x.dot(&self.w_k.t());
        let v = x.dot(&self.w_v.t());
        
        // 2. 分割成多个 head
        let q_heads = self.split_heads(&q);
        let k_heads = self.split_heads(&k);
        let v_heads = self.split_heads(&v);
        
        // 3. 计算注意力
        let mut outputs = Vec::new();
        for i in 0..self.num_heads {
            let attn = self.compute_attention(
                &q_heads[i],
                &k_heads[i],
                &v_heads[i],
            );
            outputs.push(attn);
        }
        
        // 4. 拼接并输出
        self.concat_heads(&outputs).dot(&self.w_o.t())
    }
    
    fn compute_attention(&self, q: &Array2<f32>, k: &Array2<f32>, v: &Array2<f32>) -> Array2<f32> {
        let d_k = q.shape()[2] as f32;
        
        // Q × K^T / √dk
        let scores = q.dot(&k.t()) / d_k.sqrt();
        
        // Softmax
        let weights = softmax(&scores);
        
        // × V
        weights.dot(v)
    }
}
```

### 5.5 多头注意力

多个注意力头并行计算，每个头关注不同的模式：

- 头1：语法关系
- 头2：语义关系
- 头3：位置关系
- ...

## Day 6: Transformer 架构

### 6.1 Transformer 整体架构

```
┌─────────────────────────────────────────────┐
│                  输入                        │
└─────────────────┬───────────────────────────┘
                  ↓
┌─────────────────────────────────────────────┐
│              Token Embedding                 │
│              + Position Embedding            │
└─────────────────┬───────────────────────────┘
                  ↓
┌─────────────────────────────────────────────┐
│           × N 个 Transformer Layer           │
│  ┌───────────────────────────────────────┐  │
│  │  1. Multi-Head Attention (残差+LN)    │  │
│  │  2. Feed-Forward (残差+LN)           │  │
│  └───────────────────────────────────────┘  │
└─────────────────┬───────────────────────────┘
                  ↓
┌─────────────────────────────────────────────┐
│                 Layer Norm                   │
│                 Linear                       │
│                 Softmax                      │
└─────────────────┬───────────────────────────┘
                  ↓
┌─────────────────────────────────────────────┐
│                  输出                        │
└─────────────────────────────────────────────┘
```

### 6.2 核心组件

**1. 前馈网络 (FFN)**
```
FFN(x) = ReLU(xW1 + b1)W2 + b2
```

通常：hidden_dim = 4 × embedding_dim

**2. 残差连接 (Residual Connection)**
```
output = LayerNorm(x + Sublayer(x))
```

**3. Layer Normalization**
```
LN(x) = (x - μ) / σ × γ + β
```

### 6.3 代码实现

```rust
// src/transformer/transformer_layer.rs

pub struct TransformerLayer {
    attention: MultiHeadAttention,
    ff: FeedForward,
    ln1: LayerNorm,
    ln2: LayerNorm,
}

impl TransformerLayer {
    pub fn forward(&self, x: &Array2<f32>) -> Array2<f32> {
        // 1. Multi-Head Attention + 残差 + LN
        let attn_out = self.attention.forward(x);
        let x = self.ln1.forward(&(x + attn_out));
        
        // 2. FFN + 残差 + LN
        let ff_out = self.ff.forward(&x);
        self.ln2.forward(&(x + ff_out))
    }
}
```

### 6.4 编码器 vs 解码器

| 组件 | 编码器 | 解码器 |
|------|--------|--------|
| 注意力 | 双向 | 掩码单向 |
| 输入 | 源序列 | 目标序列 |
| 用途 | 理解 | 生成 |

## Day 7: GPT 架构

### 7.1 GPT 特点

- **仅使用解码器** (Decoder-only)
- **因果注意力** (Causal Attention)：只看到当前位置之前的词
- **单向**：从左到右生成

### 7.2 GPT 与 BERT 对比

| 特性 | GPT | BERT |
|------|-----|------|
| 架构 | Decoder | Encoder |
| 注意力 | 单向 | 双向 |
| 预训练 | 语言建模 | 掩码 MLM |
| 适用场景 | 生成 | 理解 |

### 7.3 GPT 前向传播

```rust
// src/transformer/gpt.rs

pub struct GPT {
    token_embedding: TokenEmbedding,
    position_embedding: PositionEmbedding,
    layers: Vec<TransformerLayer>,
    lm_head: Linear,
}

impl GPT {
    pub fn forward(&self, input_ids: &[usize]) -> Array2<f32> {
        // 1. 嵌入
        let mut x = self.token_embedding.forward(input_ids);
        let pos_emb = self.position_embedding.forward(input_ids);
        x = x + pos_emb;
        
        // 2. 通过所有 Transformer 层
        for layer in &self.layers {
            x = layer.forward(&x);
        }
        
        // 3. 投影到词汇表
        x.dot(&self.lm_head.weight.t())
    }
}
```

### 7.4 生成过程

**自回归生成：**
```
输入: "今天天气"
输出: "很好" (概率最高)

↓ 加入输出

输入: "今天天气很好"
输出: "。" (概率最高)

↓ 继续...

最终: "今天天气很好，适合出去游玩。"
```

## Day 16: 训练与优化

### 16.1 损失函数

**语言建模损失：**

```
Loss = -Σ log P(next_token | context)
```

### 16.2 优化器

**Adam (Adaptive Moment Estimation)**
```
m = β1 * m + (1-β1) * g
v = β2 * v + (1-β2) * g²
m_hat = m / (1-β1^t)
v_hat = v / (1-β2^t)
θ = θ - α * m_hat / (√v_hat + ε)
```

### 16.3 学习率调度

```
学习率 = base_lr * min(step^(-0.5), step * warmup^(-1.5))
```

### 16.4 梯度裁剪

```
if ||g|| > threshold:
    g = g * threshold / ||g||
```

## Day 17: 量化技术

### 17.1 什么是量化？

将高精度浮点数转换为低精度整数。

```
FP32: 3.1415926535
FP16: 3.14
INT8: 3
INT4: 3
```

### 17.2 量化类型

| 类型 | 位宽 | 压缩比 | 精度损失 |
|------|------|--------|----------|
| FP32 | 32 | 1× | 无 |
| FP16 | 16 | 2× | 小 |
| INT8 | 8 | 4× | 中 |
| INT4 | 4 | 8× | 大 |

### 17.3 量化公式

```
x_quant = round(x / scale + zero_point)
x_dequant = (x_quant - zero_point) * scale
```

## Day 18-20: KV Cache 优化

### 18.1 什么是 KV Cache？

自回归生成时，之前计算的 Key 和 Value 可以缓存复用。

### 18.2 工作原理

```
Step 1: "今天" → 计算 K1, V1，缓存
Step 2: "今天 天气" → 使用 K1,V1 + 计算 K2,V2，缓存
Step 3: "今天 天气 很好" → 使用 K1,V1,K2,V2 + 计算 K3,V3
```

### 18.3 性能提升

| 序列长度 | 传统 | KV Cache |
|----------|------|----------|
| 100 | 10000 FLOPs | 1000 FLOPs |
| 1000 | 1000000 FLOPs | 10000 FLOPs |
